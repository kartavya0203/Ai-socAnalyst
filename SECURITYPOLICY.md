# Security Policy

Thank you for helping keep **AI SOC Analyst** secure.

We take security seriously and appreciate responsible disclosure of vulnerabilities.

---

# 🛡️ Supported Versions

Currently supported versions:

| Version | Supported |
|--------|-----------|
| v1.5+  | ✅ Yes |
| Below v1.5 | ❌ No |

---

# 🚨 Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

## Preferred Method

Open a private GitHub issue or contact the maintainer directly.

Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)
- Screenshots / logs if relevant

---

# 🔒 Please Do NOT

To protect users and contributors:

- Do not publicly disclose active vulnerabilities before a fix is available
- Do not post exploit code in public issues
- Do not access systems or data you do not own
- Do not perform destructive testing

---

# ⏱️ Response Commitment

We aim to:

- Acknowledge reports promptly
- Investigate responsibly
- Provide updates when possible
- Release fixes in a reasonable timeframe

---

# 🤝 Scope

Security reports are especially helpful for:

- Authentication flaws
- API security issues
- Injection vulnerabilities
- Sensitive data exposure
- Dependency vulnerabilities
- Misconfigurations
- Automation abuse risks

---

# 🙌 Thank You

Responsible security research helps improve AI SOC Analyst for everyone.
